## 2024-03-03 - Critical: Admin API Route Authorization Bypass via Direct Access
**Vulnerability:** Several administrative API endpoints (e.g., `/api/admin/inventory`, `/api/admin/offers`, `/api/admin/reviews`, `/api/admin/settings`) relied solely on global `middleware.ts` for authentication and did not explicitly verify the admin session (`verifyAdminSession`) within their route handlers.
**Learning:** Relying exclusively on Next.js middleware for critical authorization logic introduces a single point of failure. If the middleware configuration is bypassed, modified, or incorrectly maps routes, these sensitive endpoints become accessible to unauthenticated users, leading to data manipulation or exposure.
**Prevention:** Implement defense-in-depth by enforcing explicit authorization checks at the top of every administrative API route handler (e.g., `if (!(await verifyAdminSession(request))) return 401;`), regardless of the middleware's coverage.
