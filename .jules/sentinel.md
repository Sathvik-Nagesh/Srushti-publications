## 2025-03-31 - [High] Missing Rate Limiting on Order Tracking
**Vulnerability:** The `/api/orders/track` endpoint lacked rate limiting. Because order numbers (`ORD-YYMMDD-XXXX`) have a limited entropy space (10,000 per day), an attacker could easily brute-force the endpoint to enumerate valid orders. This would expose PII (Personally Identifiable Information) such as partial addresses, order contents, and pricing details.
**Learning:** Any endpoint that allows querying data via a semi-predictable identifier must implement strict rate limiting to prevent enumeration and mass data scraping, even if the data returned seems relatively benign or "public" (like tracking status).
**Prevention:** Always apply rate limiting (e.g., using `checkRateLimit`) to public-facing query endpoints, especially those that take an ID or tracking number as input.
